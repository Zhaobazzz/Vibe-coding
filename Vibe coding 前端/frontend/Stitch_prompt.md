# Stitch prompt (English) — personal site wireframe

Use this as a starting point: paste into Stitch after you fill bracketed fields from your Feishu / PRD doc.  
Prototype = **layout + copy structure only** — **no photos, avatars, logos, or decorative images** (use gray blocks or initials in monospace if needed).

---

## Prompt (copy below the line into Stitch)

```
You are a product designer generating a LOW-FIDELITY wireframe prototype (not final UI art).

Goal: a personal portfolio / personal website for [YOUR_NAME_OR_BRAND].

Constraints:
- No photographs, no avatars, no illustrations, no stock imagery. Use neutral placeholders (rectangles, initials, "IMG") where media would go.
- Clear information hierarchy, generous whitespace, strong typographic scale.
- Desktop-first, one primary breakpoint suggestion is enough.
- Output should read like a structured site map + annotated frames (sections labeled), suitable for handoff to a developer.

Visual direction (abstract, not copying any single site):
- Take inspiration from the kind of modern developer-portfolio layouts curated on https://www.designprompts.dev/ — clean grids, restrained color (1 accent + neutrals), subtle borders, monospace for labels or metadata where it helps scannability.

Content blocks to include (replace bracketed text with your real outline from the PRD):
1) Hero: [ONE_LINE_VALUE_PROP], [SHORT_SUBTITLE], primary CTA "Contact", secondary CTA "Projects"
2) About: [2–3 short paragraphs OR bullet facts: role, location, focus areas]
3) Projects: [N] cards — each card only title + 1–2 sentence description + tech tags (text only)
4) Experience / Timeline (optional): [entries with date range + title + one line each]
5) Contact: email placeholder, links as text (GitHub / LinkedIn labels only, no icons required)

Deliverables:
- Name each frame/section.
- Note spacing rhythm (e.g., 8px grid) in annotations.
- Explicitly state "no images in prototype" in a short designer note at the top of the output.
```

---

## How this fits the lesson

1. Fill `[YOUR_NAME_OR_BRAND]` and bracketed sections from **your** step-2 doc (Feishu or local notes).  
2. Keep the **no images** line so Stitch does not invent avatars.  
3. Tweak the **Visual direction** sentence if you want a different mood (e.g., brutalist, editorial).
